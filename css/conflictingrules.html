The cascade

The order of CSS rules matters; when two rules apply that have equal specificity, the one that comes last in the CSS is the one that will be used.

Specifity

The more specific the rule the higher the chances of it getting selected. Element selectors like h1 { } are less specific than class selectors .main-title { }

There's also specifity scoring which will be discussed later

Inheritance

Elements under body { color: blue; } like a span will inherit properties of the body element unless specific styling is applied to the span e.g. span { color: black; }

Some rules will not be inherited e.g. width ... otherwise CSS would be very cumbersome to use.

Controlling Inheritance

Every css property will accept these universal attributes

inherit

Turns on inheritance by telling the element to inherit properties of its parents (cascade layers)

initial

Set value of property to its initial value

revert

Set value of property to the default value as per browser styling, acting like unset

revert-layer

Set value of property to the value of a previous cascade-layer

unset

Resets the property to its natural value, which means that if the property is naturally inherited it acts like inherit, otherwise it acts like initial.
