CSS Selectors

CSS selectors define the elements to which a set of CSS rules apply.

They include: Basic selectors, Grouping selectors, Combinators, Pseudo

Basic Selectors

  • Univresal selectors — selects all elements using * or ns|* or *|*
  • Type selectors - such as input
  • Class selectors
  • ID selectors
  • Attribute selectors

Universal selectors

Selects all elements, but can be restricted to specific namespaces e.g.

  • ns|* matches all elements in the "ns" namespace.
  •  |* matches elements without a defined namespace
  • *|* matches all elements

Note on syntax: for simple selectors *.darkmode and .darkmode is really the same thing

Element selector

p selects a paragraph tag <p> ... <p>

ID selector

#title selects an element with an attribute id="title"

Class selector

.title selects an element with an attribute class="title"

Attribute selector

Targets elements on the page with specific attributes e.g.li[class="light"] This is a span with a class "light"

From: https://developer.mozilla.org/en-US/docs/Learn/CSS/Building_blocks/Selectors

CSS attribute selectors such as a[title] {} (selecting all links with titles) can also be so specific to allow a[href="https://example.com"] { }

Matches elements based on the attributes they have e.g. a[href='https://google.com'] { color: red; }

Syntax and Examples
  1. a[href*="facebook"] { } — links containing facebook.
  2. a[class~="hide"] { } — links with possibly many classes (whitesliace separated) one of which is (exactly) hide.
  3. a[href$=".org"] { } — links ending with .org
  4. a[data|="pro"] — links a data attribute whose value is pro, or begins with pro and immediately followed by a hyphen - ... meaning both data="pro-rated" and data="pro" will satisfy this condition;
  5. a[title^="ke"] — selects all <a href="..."/> elements with titles prefixed by ke
  6. a[title^="ke" i] { } — adding the i makes the selector case insensitive but adding an s makes it case sensitive.
Examples
  • This red link is red because has a title that starts with r and ends with d, and this is the css that makes it behave this way: a[title^="r"][title$="d"] { }
  • (PRODUCT) RED is red because one of its classes is red and this is the css that makes it behave this way: a[class~="red"] { }
  • Negation is also possible like this a:not[class='green'] { }

Pseudo

There's the common pseudo-classes a:hover { } which style different states of an element. But there's the pseudo-elements p::first-line { } or p::first-letter which select a part of the element instead of the entire element itself.

Combinators

Comprises of

Descendant combinators

Uses spaces e.g. input span { color:red; } will match all span elements that are inside input elements

Child combinators

Uses > to refer to direct children e.g. input > span { } will select all spans directly under input fields

General sibling combinators

Uses ~ the squiggly, to seelct siblings e.g. input ~ span

Adjucent sibling combinators

Uses + to associate adjucent elements e.g. input + div { } selects all div elements next to input fields
