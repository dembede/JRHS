handling strings

Topic reference: https://developer.mozilla.org/en-US/docs/Learn/JavaScript/First_steps/Useful_string_methods

escaping characters

'I\'ve got nothing to worry about some_text'

concatenating strings

Template literals use backticks `` e.g. const greeting = `Niaje`

we can also add variables to greeting using ${some_text} e.g.
greeting = `Hello ${firstName}` prints out "Hello Sammy"

concatenating with + is still okay, however using template literals `` and including variables with ${} usually gives you more readable code

useful methods/tips

Number() converts anyting to a number (if it can)

toString() every number has this method

Template literals respect line breaks. e.g.
`I have a phone. {enter} its color is red.`

Getting length of a string firstName.length returns a number

Checking if a string contains a substring
firstName.includes('some_text') returns boolean
firstName.startsWith('some_text') returns boolean
firstName.endsWith('some_text') returns boolean
firstName.indexOf('some_text') returns number. +ve number if present, -ve number if absent
firstName.indexOf('some_text',"fromPosition") searches from index specified by "fromPosition". This can be useful in finding 2nd, 3rd... occurrences of the same string

extracting a substring from a string
• use the slice() method, like this slice(startIndex, endIndex)
• example: firstName.slice(1, 4); // 'ammy'
• without the endIndex it extracts the string to the end
Other options of slice() here https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/String/slice
— firstName.slice(-4); // "ammy"
— firstName.slice(-1,02); // "" since those positions don't make sense

changing case is possible e.g. firstName.toUpperCase(); // SAMMY or can use firstName.toLowerCase(); // sammy

updating parts of a string is possible e.g. firstName.replace('my','wel'); // Samwel or firstName.replaceAll('m','n') // Samnny because the former only replaces the first occurence of 'm'
