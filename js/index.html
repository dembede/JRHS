first steps - the guessing game

The assignment reads:
I want you to create a simple guess the number type game. It should choose a random number between 1 and 100, then challenge the player to guess the number in 10 turns. After each turn the player should be told if they are right or wrong, and if they are wrong, whether the guess was too low or too high. It should also tell the player what numbers they previously guessed. The game will end once the player guesses correctly, or once they run out of turns. When the game ends, the player should be given an option to start playing again.

We have selected a random number between 1 and 100. See if you can guess it in 10 turns or fewer. We'll tell you if your guess was too high or too low.

This incorporates learning about:

  • functions — e.g. checkGuess()
  • operators — they allow us to perform tests, do math, join strings together, and other such things.
    • arithmetic operators — + - * /
    • comparison operators — === != < >
  • conditionals — see the checkGuess() { ... } function in script.js.
  • events — would look something like guessSubmit.addEventListener('click', checkGuess);

storing information - variables

Using var had its advantages e.g. variable hoisting - where variables can be initialised before being declared, but its disadvantages surpass those suffered by let and const.

As a principle, use const when you can, and only use let when you have to.

naming conventions
  • Always better to stick to camelCase
  • Don't use underscores before as a prefix - these are reserved for other JS functions
variable types

JavaScript is a "dynamically typed language", which means that, unlike some other languages, you don't need to specify what data type a variable will contain (numbers, strings, arrays, etc).
