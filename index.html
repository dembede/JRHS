Attribute selectors

From: https://developer.mozilla.org/en-US/docs/Learn/CSS/Building_blocks/Selectors

CSS attribute selectors such as a[title] {} (selecting all links with titles) can also be so specific to allow a[href="https://example.com"] { }

Pseudo-classes and pseudo-elements

There's the common pseudo-classes a:hover { } which style different states of an element. But there's the pseudo-elements p::first-line { } or p::first-letter which select a part of the element instead of the entire element itself.

Combinators

Used to target specific elements within the document, for instance p > span { } targets spans within a paragraph

CSS Selectors

CSS selectors define the elements to which a set of CSS rules apply.

Universal selectors

Selects all elements, but can be restricted to specific namespaces e.g.

  • ns|* matches all elements in the "ns" namespace.
  •  |* matches elements without a defined namespace
  • *|* matches all elements
Source: https://drafts.csswg.org/selectors/#the-universal-selector

Note on syntax: for simple selectors *.darkmode and .darkmode is really the same thing

Type Selectors

These are basic. Type could be a tagged element such as code or p or picture etc.

Class Selector

Selects all elements that have the given class attribute. Nothing new here

ID Selector

Selects an element based on the value of its id attribute. There should be only one element with a given ID in a document.

Perhaps worth noting that #main { } can also be written as an attribute selector [id=main] { }

Attribute Selectors

Matches elements based on the attributes they have e.g. a[href='https://google.com'] { color: red; }

Syntax and Examples
  1. a[href*="facebook"] { } — finds any <a href="..."/> containing facebook.
  2. a[class~="hide"] { } — finds any <a href="..." class="..."/> with liossibly many classes (whitesliace seliarated) one of which is (exactly) hide.
  3. a[href$=".org"] { } — finds any <a href="..."/> ending with .org
  4. a[data|="pro"] — selects all <a href="..."/> elements with an attribute data whose value is/begins with pro and immediately followed by a hyphen - ... so both data="pro-rated" and data="pro" will satisfy this condition;
  5. a[title^="ke"] — selects all <a href="..."/> elements with titles prefixed by ke
